# LADF v2.1.0-alpha Release Notes

> **Release type:** Pre-production (alpha)

LADF v2.1.0-alpha centers on the KPI variant system, strengthening presets, heuristics, and extensibility so KPI cards stay expressive even with sparse configs.

## Highlights

* **KPI variant preset library** covers general subvariants (currency, large-value, percentage, ratio, hours, duration, negative) plus variant-specific presets for icon (rating/alert/capacity/velocity) and compact (growth/minimal/score/efficiency). Presets layer base defaults (icons/badges), respect explicit overrides, and normalize aliases like minimal/halo/backdrop/compact into supported variants.
* **Format and value resolution** add `valueFrom` control (first/last row), suffix support (`valueSuffix`/`unit`), ratio numerator/denominator resolution from encodings, and broader option aliasing across trend, ratio, and sparkline keys.
* **Trend chip pipeline** infers direction/tone/icon automatically, honors explicit `trendChip*` options, and can derive trend values from sparkline deltas when no trend field is provided; decimal precision is configurable.
* **Sparkline integration** accepts inline arrays, keyed arrays, or derivation from KPI data (`sparklineFromData`), reuses `sparklineValueKey` for mixed row shapes, and feeds tone/direction plus fallback value when the primary value is absent.
* **Compact badge treatment** confines badges to the compact variant with tone/icon presets, keeping other variants uncluttered.
* **KPI variant extensibility** exposes `kpiVariantRegistry` and `registerKpiVariant` so custom KPI renderers can be registered alongside the built-ins.

## Breaking Changes

* Badge text now renders only on the `compact` variant; other variants ignore badge props.
* `valueFrom` defaults to the last row when `sparklineFromData` is enabled (first row otherwise), which may change the displayed value for multi-row KPI inputs.
* Unknown variant ids normalize to the `clean` preset, so unsupported names now fall back instead of creating unmatched classes.

## Notes

* This remains an **alpha** build intended for **pre-production** environments.
* See `docs/lazy-dashboard-option-coverage.md` for the updated option matrix covering KPI controls and editor parity.
* Feedback is especially welcome on the new KPI presets, trend/sparkline heuristics, and badge behavior before stabilizing for beta.
